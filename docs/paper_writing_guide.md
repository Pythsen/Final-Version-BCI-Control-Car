# 论文写作建议（面向本项目）

## 1. 题目可选

- 基于运动想象 EEG 的实时脑控小车系统：从离线训练到在线闭环控制
- A Real-Time Motor-Imagery EEG Controlled Car with Causal Filtering and Distilled Lightweight Model

## 2. 摘要模板

建议 4 句结构：

1. 背景与问题：MI-EEG 在线控制面临实时性与准确率权衡。  
2. 方法：提出「离线训练 + 因果滤波在线推理 + 串口控制」闭环方案。  
3. 结果：报告离线交叉验证与在线会话准确率、延迟、稳定性。  
4. 结论：说明在嵌入式/低算力平台上的可部署价值。

## 3. 论文主体结构

### 3.1 Introduction

- 应用场景：辅助驾驶、康复、人机交互。  
- 现有问题：离线高精度方法难直接迁移到在线实时控制。  
- 贡献点（建议写 3 条）：
  - 完整端到端闭环（采集/发流/推理/控制）。
  - 兼顾精度与部署的轻量模型（Student）+ 蒸馏训练（Teacher）。
  - 因果滤波在线管线，避免离线滤波在实时场景中的信息泄漏。

### 3.2 Methods

- 数据集：BCI Competition IV 2a（9 被试、4 类 MI）。  
- 预处理：0.5-100 Hz 带通 + 50 Hz 陷波；离线 `filtfilt` / 在线 `lfilter`。  
- 特征与模型：原始时序分支 + CSP 分支融合；Student 与 Teacher 结构。  
- 训练策略：K-fold、类别权重、早停、蒸馏损失（含温度与权重）。  
- 在线系统：Sender/Receiver 协议、Cue 触发逻辑、串口动作映射。

### 3.3 Experiments

- 离线评估：被试混合 K-fold 准确率、每类 Recall/F1。  
- 在线评估：按 trial 报告准确率与混淆矩阵。  
- 实时性：端到端平均推理时延、控制触发延迟。  
- 稳定性：长时段运行丢包率/超时保护触发次数。

### 3.4 Results

建议至少放 3 张表/图：

- 表 1：不同版本（v42/v65/v50pro）离线指标对比。  
- 表 2：在线 trial 级结果（Accuracy、Macro-F1、每类召回）。  
- 图 1：系统结构图（数据流 + 控制流）。  
- 图 2：典型在线会话时间轴（Cue、预测、动作）。

### 3.5 Discussion

- 为什么在线性能与离线有差距（因果滤波、时序噪声、跨会话偏移）。  
- 系统优势（闭环可部署）与局限（跨被试泛化、硬件依赖）。  
- 未来工作（自适应校准、在线增量学习、更多控制指令）。

### 3.6 Conclusion

一段总结：

- 明确“提出了什么系统”。  
- 明确“在何种设置下达到何种效果”。  
- 明确“可复现性与应用前景”。

## 4. 结果呈现建议（可直接套用）

- 统计口径要固定：trial-level / session-level 不混用。  
- 所有结果附标准差或置信区间。  
- 报告显著性检验（如与 baseline 对比的配对检验）。  
- 线上结果附具体会话日期与被试编号。

## 5. 复现清单（附录建议）

- 代码结构说明（`src/`, `models/`, `outputs/`）。  
- 关键参数表（窗口长度、滤波参数、蒸馏超参数）。  
- 运行命令清单（训练、时间线生成、在线仿真、评估）。  
- 硬件配置（采样率、串口参数、底盘控制协议）。
